Аналіз правдивості

Аналізує текст без зовнішніх API. Штрафи:
  • Хедж-слова (allegedly, reportedly, sources say...) → −3 за слово (макс −15)
  • Клікбейт у заголовку (shocking, exposed, bombshell...) → −5 за патерн (макс −10)
  • Емоційна лексика (catastrophe, outrage, rigged...) → −2 за слово (макс −10)
  • Маніпулятивні патерни (fake news, caught red-handed...) → −8 за патерн (макс −16)
  • Конспірологія (false flag, deep state, flat earth...) → −20 фіксовано + cap 40 для всього score
Витягує ключові слова із заголовку і шукає тему у трьох API: NewsAPI, The Guardian, GNews. Підтвердження: 0 джерел → 0 | 1 → 10 | 2 → 22 | 3 → 35. Поріг: мінімум 3 результати щоб уникнути рахування статей-спростувань. Автоматично вимикається для статей з датасету (2016-2017 рр.).
Три підшари:
  • Репутація домену (0–12): reuters.com, bbc.com → HIGH(12); nytimes.com → GOOD(9); невідомий → UNKNOWN(5); infowars.com → LOW(0). Всі .gov і .edu → HIGH.
  • ClaimBuster (0–10): власна regex-реалізація. Рахує кількість числових тверджень, абсолютів (never/always), дієслів заяв (claimed/confirmed). 0–1 сигнал → 10; 2–3 → 6; 4+ → 2.
  • RSS-перевірка (0–8): порівняння з RSS-стрічками Snopes та PolitiFact. TRUE → 8; не знайдено → 6; FALSE → 0.
Лексикон AFINN-111 (~150 слів). Кожному слову присвоєно оцінку від −5 до +5. Нормалізований результат: −1.0 (дуже негативний) ... +1.0 (дуже позитивний). Мітки: POSITIVE (>0.1) / NEUTRAL / NEGATIVE (<−0.1). Не впливає на credibility score.
Формула Flesch Reading Ease: 206.835 − 1.015×(слів/речень) − 84.6×(складів/слів). Рівні: VERY_EASY (90–100) → EASY → STANDARD → DIFFICULT → VERY_DIFFICULT (0–30). Не впливає на credibility score.
